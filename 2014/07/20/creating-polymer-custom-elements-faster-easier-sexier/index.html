Creating Polymer custom elements faster, easier & sexier

Summary: I made a sweet-ass factory for creating Polymer elements with Slim, Sass, Coffeescript and BrowserSync, served with Gulp. You can get it here.

It's no surprise that I have an odd tendency to find myself stumbling along the bleeding edges of web technologies, trying out almost a little too much. "You collect technologies like others collect Pokémon cards!" is something I hear often (ahem Lasse!)

I can't help it though, I love learning new things. And my newest addition to my "deck of techs" is Polymer - a solution that enables the use of custom web components or "elements" within web projects. After my initial run-ins I don't doubt that web components (not necessarily Polymer per se) will bring on a tectonic shift in the landscape of the web in the coming years, and isn't it awesome that you can already play with it today? I knew you'd think so too!

But...

Creating Polymer elements is quite straightforward out-of-the-box, but it does take the usual "create folder, install bower and npm, create the typical hello-world element, refresh browser, check, refresh browser, check et cetera"-process to really get started.

In short: It's okay when still playing in the sandbox, but I want a sexier way of going from idea to crafting an element to releasing it.

Introducing "Abathur"

Yes, it's a Starcraft reference, and yes it's totally the right name for something that literally spins Polymer elements like it was the easiest thing in the world. By allowing you to use Slim, Sass and CoffeeScript to craft the element and then automatically get it compiled into .html, .min.css and .min.js via Gulp is awesome10th. But let's take it to the 11th and throw in live browser reloading and synchronization between multiple browsers and devices with BrowserSync. Boom!
