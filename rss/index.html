What have you been crafting today? Much? Nothing? In any case, if you have 180 seconds to waste anyway, I just wanted to share a little list of features that I've come to like about Adobe's "comp-to-code" web application called "Project Parfait".

Yep, pop in that PSD and get some nifty information needed for turning that static file into something magic. Pretty simple, but it's nice to see some software that seamlessly accepts a PSD without being a pain in the ass. So without any further ado, here's the top 4 features I like about Project Parfait:

1. Typographic Overviews

That's right, after uploading the PSD, you can click on text anywhere in the document and see its font family (if on TypeKit), weights and what sizes are being used where. A very nice feature in itself, but what I didn't expect is the ability to switch between different font units. I usually develop in rem, so being able to switch to that unit is really handy (for me, at least).

2. Element Distance Calculations

Project Parfait supports clicking and then shift-clicking on two elements which will give you the x and y distance (in pixels only, it seems) between the two. I've received several PSDs from a graphic designer and sometimes ended up pixel pushing things until it looked right visually, only to eventually jump into the PSD and manually using the ruler tool to calculate the distance. No need for that anymore!

3. Color Palette Extraction (!)

It might seem like a banality, but if I had a dollar for every time I had to either a.) extract a color palette manually because no one supplied me with any web-friendly values or b.) email clients or associated designers for those values I'd finally be able to get that ruby-covered gold scepter I've always wanted... Project Parfait allows you to quickly extract those values and have them ready-at-hand. I like that.

4. CSS Code Preview

This is a no-brainer, but incredibly helpful; click on an element and you will get a (well-)generated CSS code for that element in a previewer window. I know it's simple, but ignoring the width and height code, it makes it very easy to just click, copy and paste into your stylesheet. My only concern is how this fits a workflow involving pre-processors such as Sass or Less. It could be interesting to see if Project Parfait could understand a specific naming convention of a PSDs folder structure that would then translate into neatly organized components... or something along those lines.

Aight, everything must come to an end, Project Parfait has more features to offer, and you should give it a little checkout when you get the time. I'll definitely try giving it a spin into my workflow.

Keep awesome!

You can learn more about the Ghost blogging platform here, but in this article I'll cover how to get a Ghost blog up and running at the cost of absolutely nothing. Zero, null, nuffin' (if that's even a word).

All you need is a GitHub Account, a cup of coffee and a little patience.

Step one, prepare a GitHub repository

After creating your GitHub account, follow GitHub's own guide on how to prepare GitHub Pages for your repository.

You should then end up with a repository with an URL of something like:

GitHubUsername.github.io

Also, if you want to use a custom domain (highly recommended!), be sure to add a CNAME file (in uppercase!) to your repository. You can do this by going to your repository and click on the + symbol next to your repository's name (in blue). Just name the file CNAME and input the domain you own here.

Notice: I had to write www.andersschmidt.com even though some guides tell you can write the root version AKA the-one-without-the-www. After that I just forwarded andersschmidt.com to the www-version.

Setting up the DNS

With the CNAME file in place, go to your DNS or the company that hosts your domain name and add a CNAME value where the host is www and the destination is GitHubUsername.github.io. I set my TTL to 10 minutes, but you can keep it at 1 hour if you want.

Step two, install Ghost

Go here for downloads specific to your operating system (I use Linux / ElementaryOS), download Ghost and follow the installation instructions.

I have my Blog folder inside Dropbox for extra backup, but your mileage may vary. After installing Ghost and confirming that it does actually run, it's time to bring it to GitHub via a sweet piece of code named Buster.

Step three, install Buster

Buster takes your Ghost blog and turns it into a static site in a format that GitHub likes, meaning that it is possible to bring your blog to the repository you made earlier. Neat huh? And since the repository is free, so is your blog!

To install Buster, you first need Pip and how easy it is to install depends on your operating system (on Linux it's easy). After installing Pip you can then run the following in your terminal or console:

pip install buster

Remember to have wget installed beforehand, since it's required if you want to use Buster. Most Linux versions have wget by default, but if you're on a Mac, you can run:

brew install wget

After Buster is installed, navigate to your Ghost blog's folder and run buster setup to prepare things. You will have to input the URL to your GitHub repository you made earlier for this to work.

Hereon it's pretty straightforward. After starting up Ghost via npm start, go to http://127.0.0.1:2368/ghost/ and begin writing what you want, preview it locally and when you're ready to push it online, do the following:

buster generate --domain=http://127.0.0.1:2368 && buster deploy

You will be prompted to input your GitHub credentials and if things are correct Buster will push the static site version of your blog to your GitHub repository and prezto, your blog is now live! For free. Isn't that nifty?

Optional step: faster deployment

If you're on a NIX-based system like Linux or Mac OS, it is possible to make it less annoying to generate and deploy your Ghost site. I personally think it's tedious to copy and paste the above command every time, so I made a small BASH script you can add to your .bashrc file to speed things up a little:

openGhost(){
  cd ~/Your/Blog/Folder/;
  npm start;
}
alias ghost=openGhost;
alias ghostup="sudo buster generate --domain=http://127.0.0.1:2368 && sudo buster deploy";

The ghost command will, if you've correctly changed the Your/Blog/Folder to match the folder your Ghost blog lives in, neatly start up Ghost from wherever you are in your system and the ghostup command will simply run the previous command you had to copy and paste.

It's not much, but if you write a lot it's a neat timesaver.

In closing, enjoy blogging with Ghost for free and help spread the love!
